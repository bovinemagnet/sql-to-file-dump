= sql-to-file-dump
:description: Overview of the sql-to-file-dump JDBC export CLI

`sql-to-file-dump` is a Quarkus 3.36.2 + Picocli command-line tool (command name `jdbc-export`) that connects to any JDBC-compatible database, runs a read-only SQL query, and streams the results to a file in your chosen format.
It is designed for reliable, memory-efficient bulk data export with no surprises.

== Features

* *Streaming output* — rows are written directly from the JDBC `ResultSet` to disk; the full result set is never buffered in memory
* *Five output formats* — JSON, NDJSON, CSV, TSV, and Parquet
* *Read-only safety* — only `SELECT` and `WITH` statements are accepted; `SqlSafetyValidator` rejects anything else, guarding against accidental mutation
* *Sluice daemon ops console* — a browser-based operations console for submitting and monitoring export jobs; see xref:daemon-dashboard.adoc[Sluice Daemon Dashboard]
* *Structured exit codes* — exit codes 0–6 and 99 allow reliable scripting and CI integration
* *Bundled JDBC drivers* — PostgreSQL and Oracle drivers are included; no separate driver installation is required
